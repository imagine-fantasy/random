[2:45 — 4:00] Architecture
Show diagram.
"What you just saw is Phase 1. Here's the full system we've designed.
A pipeline failure triggers the orchestrator. It routes by failure type and manages the agent lifecycle using a sequential workflow.
The log triage agent diagnoses. The code reader agent clones the repo and reads our firm's custom Terraform modules using semantic search — understanding what the module should do versus what the error says.
Then the fix agent enters an agentic loop. It generates a fix, runs a local plan or build, validates, iterates if errors remain — until the build is clean. Then hands off to the PR agent."

[4:00 — 4:30] Why this matters
"Terraform is a particularly strong use case because we use custom modules that don't behave like standard providers. Bugs surface at runtime, not at review. We approve the PR, everything looks correct, the apply fails three days later. Nobody currently flags that gap. The PR agent will — marking every fix with whether it was a runtime issue or a review gap. Visibility into where the process broke, not just what broke."

[4:30 — 5:00] Close
"The numbers are simple. We lose 30 to 60 minutes every time a pipeline fails just on diagnosis alone. With this agent — under 2 minutes. Phase 1 is built and running today. The full loop — code reader, fix agent, PR automation, Teams and email notification — that's the roadmap we've laid out.
We're not replacing engineers. We built this to replace the parts of the job nobody wants to do. Thank you."
